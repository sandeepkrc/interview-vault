Decorators in Python

@decorator — Modify or enhance functions without changing their source code.
A powerful metaprogramming technique for reusable functionality.

📖 What is a Decorator? (Theory)

A decorator is a function that takes another function as an argument, extends or modifies its behavior, and returns a new function — without permanently modifying the original function. Decorators are applied using the @decorator_name syntax.

Core Concept: Functions in Python are first-class citizens — they can be passed as arguments, returned from other functions, and assigned to variables. Decorators leverage this property to wrap functions with additional logic.
Basic Decorator Syntax 
def my_decorator(func):
    def wrapper():
        print("Something before the function")
        func()
        print("Something after the function")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
# Output:
# Something before the function
# Hello!
# Something after the function
Output: 
Something before the function
Hello!
Something after the function

🎯 Why Use Decorators? (Use Cases)

⏱️ Timing / Logging

Measure function execution time or log calls for debugging.

🔐 Authentication / Authorization

Check user permissions before executing protected functions.

💾 Caching (Memoization)

Cache expensive function results to improve performance.

✅ Input Validation

Validate arguments before function execution.

🔄 Retry Logic

Automatically retry failed operations (API calls, DB queries).

📝 DRY Principle

Eliminate repetitive code across multiple functions.

⏱️ Example 1: Timing Decorator

measure execution time 
import time

def timer_decorator(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"⏱️ {func.__name__} took {end - start:.4f} seconds")
        return result
    return wrapper

@timer_decorator
def slow_function():
    time.sleep(1)
    return "Done!"

print(slow_function())
Output: 
⏱️ slow_function took 1.0012 seconds
Done!

🔐 Example 2: Authentication Decorator

login_required decorator 
def login_required(func):
    def wrapper(user, *args, **kwargs):
        if not user.get('is_authenticated', False):
            return "❌ Access Denied! Please login first."
        return func(user, *args, **kwargs)
    return wrapper

@login_required
def dashboard(user):
    return f"✅ Welcome {user['name']} to your dashboard!"

# Testing
user1 = {'name': 'Alice', 'is_authenticated': True}
user2 = {'name': 'Bob', 'is_authenticated': False}

print(dashboard(user1))
print(dashboard(user2))
Output: 
✅ Welcome Alice to your dashboard!
❌ Access Denied! Please login first.

🔄 Example 3: Retry Decorator (Error Handling)

retry on exception 
def retry(max_attempts=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"⚠️ Attempt {attempt} failed: {e}")
                    if attempt == max_attempts:
                        raise
            return None
        return wrapper
    return decorator

@retry(max_attempts=3)
def unstable_api_call():
    import random
    if random.random() < 0.7:
        raise Exception("API temporarily unavailable")
    return "🎉 API Success!"

print(unstable_api_call())
Output (example): 
⚠️ Attempt 1 failed: API temporarily unavailable
⚠️ Attempt 2 failed: API temporarily unavailable
🎉 API Success!

💾 Example 4: Caching / Memoization Decorator

cache expensive computations 
def cache(func):
    cache_storage = {}
    def wrapper(*args):
        if args in cache_storage:
            print(f"📦 Cache hit for {args}")
            return cache_storage[args]
        print(f"⚙️ Computing result for {args}")
        result = func(*args)
        cache_storage[args] = result
        return result
    return wrapper

@cache
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

print(fibonacci(10))
print(fibonacci(10))  # cached call
Output: 
⚙️ Computing result for (10,)
55
📦 Cache hit for (10,)
55

⚙️ Decorators with Arguments (Parameterized)

parameterized decorator 
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(times=3)
def greet(name):
    print(f"Hello, {name}!")

greet("Pythonista")
Output: 
Hello, Pythonista!
Hello, Pythonista!
Hello, Pythonista!
Important: Parameterized decorators have an extra layer — the outer function accepts arguments, then returns the actual decorator.

🏷️ Preserving Function Metadata with @wraps

using functools.wraps 
from functools import wraps

def my_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        """Wrapper docstring"""
        print("Before")
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def important_function():
    """This is an important function"""
    pass

print(important_function.__name__)
print(important_function.__doc__)
Output: 
important_function
This is an important function
Without @wraps, the function name and docstring would be replaced by the wrapper's — breaking introspection and debugging tools.

⛓️ Chaining Multiple Decorators

stacking decorators 
def bold(func):
    def wrapper():
        return f"<b>{func()}</b>"
    return wrapper

def italic(func):
    def wrapper():
        return f"<i>{func()}</i>"
    return wrapper

@bold
@italic
def greet():
    return "Hello"

print(greet())  # <b><i>Hello</i></b>
Output: 
<b><i>Hello</i></b>
Decorators = Higher-Order Functions + Closures
They enable clean, reusable, and maintainable code by separating cross-cutting concerns.
