📘 Non-Static Methods in Python

Instance Methods · Class Methods · Property Methods (Non-Static Only)

🔵

Instance Method

@ No decorator needed
Instance methods are the most common type. They take self as the first parameter and can access or modify both instance attributes and class attributes. 
class Student:
    school = "Greenwood High"  # class attribute
    
    def __init__(self, name, grade):
        self.name = name          # instance attribute
        self.grade = grade
    
    def get_details(self):       # instance method
        return f"{self.name} - Grade {self.grade} | {self.school}"
    
    def promote(self):
        self.grade += 1
        return f"🎉 {self.name} promoted to grade {self.grade}!"

# Creating instance and calling methods
student1 = Student("Emma", 10)
student2 = Student("Liam", 9)

print(student1.get_details())
print(student2.get_details())
print(student1.promote())
▶️ OUTPUT
Emma - Grade 10 | Greenwood High
Liam - Grade 9 | Greenwood High
🎉 Emma promoted to grade 11!
  • ✅ First parameter: self (refers to instance)
  • ✅ Can access both instance & class attributes
  • ✅ Can modify object state
  • ✅ Called via: object.method()
🟣

Class Method

@classmethod decorator
Class methods receive the class (cls) as the first parameter. They can modify class state that applies to all instances. Often used as factory methods. 
class Company:
    total_employees = 0
    company_name = "TechCorp"
    
    def __init__(self, name):
        self.employee_name = name
        Company.total_employees += 1
    
    @classmethod
    def change_company_name(cls, new_name):
        old_name = cls.company_name
        cls.company_name = new_name
        return f"🏢 Name changed: {old_name} → {new_name}"
    
    @classmethod
    def from_string(cls, emp_str):    # factory method
        name = emp_str.split("-")[1]
        return cls(name)

# Using class methods
print(f"Initial: {Company.company_name}")
print(Company.change_company_name("InnovateSoft"))

emp = Company.from_string("ID-123-Sarah")
print(f"Created employee: {emp.employee_name}")
print(f"Company now: {Company.company_name}")
▶️ OUTPUT
Initial: TechCorp
🏢 Name changed: TechCorp → InnovateSoft
Created employee: Sarah
Company now: InnovateSoft
  • ✅ First parameter: cls (refers to class)
  • ✅ Can modify class attributes across all instances
  • ✅ Alternative constructors (factory methods)
  • ✅ Called via: Class.method() or instance.method()
💎

Property Method

@property | @getter | @setter
Property methods allow you to define methods that behave like attributes. They provide controlled access to instance attributes with validation logic. 
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self._balance = balance    # protected attribute
    
    @property
    def balance(self):           # getter
        return f"${self._balance:,.2f}"
    
    @property
    def account_summary(self):   # computed property
        return f"Account of {self.owner}: {self.balance}"
    
    @balance.setter
    def balance(self, amount):    # setter with validation
        if amount < 0:
            raise ValueError("Balance cannot be negative!")
        self._balance = amount
        print(f"✅ Balance updated to ${amount:,.2f}")

# Using property methods like attributes
account = BankAccount("Alice", 1000)
print(account.account_summary)     # property accessed without ()
print(account.balance)              # getter
account.balance = 2500              # setter
print(account.balance)
▶️ OUTPUT
Account of Alice: $1,000.00
$1,000.00
✅ Balance updated to $2,500.00
$2,500.00
  • ✅ Methods that behave like attributes (no parentheses)
  • ✅ Encapsulation with getter/setter logic
  • ✅ Computed / derived properties
  • ✅ Validation and data integrity
💡 Note: Properties are non-static because they operate on instance data through getter/setter methods.

🔍 Key Takeaway: Instance, Class, and Property methods are all non-static because they interact with instance or class state. Static methods (@staticmethod) are the only ones that don't receive self or cls.
