Class Methods Deep Dive

Instance · Static · Class · & more — Python OOP explained with real examples

Understand the difference between instance methods, static methods, and class methods in Python, plus a bonus look at property methods. Each includes clear code & terminal-like output.

Instance Method self
Bound to an object instance. Can access & modify instance attributes, and class attributes via self.__class__. Most common method type. 
class Book:
    def __init__(self, title, author):
        self.title = title
        self.author = author

    def describe(self):          # instance method
        return f"📖 '{self.title}' by {self.author}"

# usage
my_book = Book("Atomic Habits", "James Clear")
print(my_book.describe())
Sample Output
📖 'Atomic Habits' by James Clear
Instance method works on specific object data.
  • First parameter = self
  • Can read/write instance state
  • Called via instance: obj.method()
Static Method @staticmethod
Utility function inside a class. No self or cls. Cannot access instance/class attributes directly — pure logic. 
class MathUtils:
    @staticmethod
    def is_even(number):
        return number % 2 == 0

    @staticmethod
    def add(a, b):
        return a + b

# called directly on class
print(MathUtils.is_even(7))   # False
print(MathUtils.add(15, 27))  # 42
Output
False
42
  • No implicit first parameter
  • Group related functions inside class
  • Called on class or instance
Class Method @classmethod
Bound to the class, not the instance. Receives cls (class) as first parameter. Used for factory methods or modifying class state. 
class Car:
    wheels = 4                     # class attribute
    total_cars = 0

    def __init__(self, model):
        self.model = model
        Car.total_cars += 1

    @classmethod
    def update_wheels(cls, new_count):
        cls.wheels = new_count

    @classmethod
    def from_string(cls, car_string):
        model = car_string.split("-")[1]
        return cls(model)

# usage
Car.update_wheels(6)
print(f"Wheels: {Car.wheels}")
Output
Wheels: 6
  • First param = cls (the class itself)
  • Can modify class attributes
  • Works as alternative constructors
Property & More @property
Special methods: @property (getter/setter), @abstractmethod, dunder methods. They extend OOP capabilities. 
class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @property
    def fahrenheit(self):
        return (self._celsius * 9/5) + 32

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("Too cold!")
        self._celsius = value

temp = Temperature(25)
print(temp.fahrenheit)   # 77.0
Property output
77.0
  • @property → computed attributes
  • Dunder methods: __str__, __len__
  • Abstract methods for interfaces

🐍 All-in-One Python Class

Below class Employee demonstrates instance method, class method, and static method working together. Watch how class state changes.

class Employee:
    company = "TechCorp"          # class attribute
    emp_count = 0

    def __init__(self, name, salary):
        self.name = name
        self.salary = salary
        Employee.emp_count += 1

    # instance method
    def get_info(self):
        return f"👤 {self.name} | Salary: ${self.salary} | Company: {self.company}"

    # class method - modify company name for all instances
    @classmethod
    def change_company(cls, new_name):
        old = cls.company
        cls.company = new_name
        return f"🏢 Company renamed: '{old}' ➝ '{new_name}'"

    # static method - utility check
    @staticmethod
    def is_high_salary(salary_amount):
        return salary_amount > 120_000

# ----- demo & output -----
e1 = Employee("Alice Chen", 95000)
e2 = Employee("Bob Rivera", 145000)

print(e1.get_info())
print(e2.get_info())
print(f"High salary? Bob: {Employee.is_high_salary(145000)}")
print(Employee.change_company("InnovateX"))
print(e1.get_info())   # reflects new company name
print(f"Total employees: {Employee.emp_count}")
▶️ EXECUTION OUTPUT
👤 Alice Chen | Salary: $95000 | Company: TechCorp
👤 Bob Rivera | Salary: $145000 | Company: TechCorp
High salary? Bob: True
🏢 Company renamed: 'TechCorp' ➝ 'InnovateX'
👤 Alice Chen | Salary: $95000 | Company: InnovateX
Total employees: 2
Insight: change_company (class method) updates class attribute for all instances; is_high_salary static method works standalone; instance method accesses both instance and class data.

Quick comparison: Instance vs Static vs Class Methods

Instance

👤 First param: self
✅ Accesses instance attributes
🔁 Modifies object state
📌 obj.method()

Static

⚙️ No self/cls
🧮 Utility / helper logic
🚫 Cannot modify class/instance
📌 Class.method()

Class

🏷️ First param: cls
🔄 Modifies class state
🧬 Factory methods (alternative constructors)
📌 Class.method() or instance

Property

✨ Getter/setter behavior
🧠 Attribute-like access with logic
📌 @property decorator

📌 Choose the right method

  • ✅ Instance method → operate on specific object data, need access to self attributes.
  • ✅ Static method → pure function that logically belongs to class, doesn't touch class/instance data.
  • ✅ Class method → need to work with class variables or create instances in an alternative way (polymorphism friendly).
  • ✅ Property → encapsulate attribute access with validation/computed logic.

Clean & expressive OOP

Calling rules: Instance method → only from instance (obj.method()), Static & Class → both class and instance work.
Decorators: @staticmethod , @classmethod , @property
