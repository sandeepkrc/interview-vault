Live Code Example: References & Identity 
import sys
import gc

# 1. Immutability & interning (small integers)
a = 100
b = 100
print("a is b:", a is b)          # True (cached small int)

c = 1000
d = 1000
print("c is d:", c is d)          # False (no interning beyond range)

# 2. Reference counting demonstration
my_list = [1, 2, 3]
ref_count = sys.getrefcount(my_list) - 1   # minus temporary ref
print(f"Reference count of my_list: {ref_count}")

another_ref = my_list
print(f"After new ref: {sys.getrefcount(my_list)-1}")

# 3. Object id() – memory address (simulated)
print(f"Memory address (id) of my_list: {id(my_list)}")

# 4. Garbage collector trigger (demo)
del another_ref
print("After deletion, refcount decreased")

Key Insight: Variables on the stack store memory addresses (pointers) to objects stored in the heap. The Python memory manager automatically allocates heap space for objects.

📌 Garbage Collection (GC)

Python uses reference counting as primary mechanism, plus a generational garbage collector to detect and collect reference cycles (circular references).

gc.collect() → manually trigger collection.
gc.get_threshold() → (700, 10, 10) generation thresholds.

🔢 Reference Counting

Each object tracks how many references point to it. When count reaches 0, memory is freed immediately.

import sys
x = []
print(sys.getrefcount(x))  # includes temporary ref

🧠 Small Object Allocator

Python uses memory pools (arenas) for small objects to reduce fragmentation and improve performance.

PyMalloc → optimized allocator for objects < 512 bytes.
Memory Management with Custom Class 
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age
    def __del__(self):
        print(f"🗑️ Deleting {self.name} from memory")

# Create objects
p1 = Person("Alice", 28)
p2 = Person("Bob", 32)
p3 = p1   # new reference to Alice

print("Ref count of p1:", sys.getrefcount(p1)-1)
del p1
print("After deleting p1, p3 still references Alice")
del p3
print("Now both references removed — Alice object becomes garbage")
Ref count of p1: 2
After deleting p1, p3 still references Alice
Now both references removed — Alice object becomes garbage
🗑️ Deleting Alice from memory
The __del__ method is called when reference count drops to 0 (immediately).
String Interning & Memory Optimization 
s1 = "hello_world"
s2 = "hello_world"
print(s1 is s2)   # True (interned by interpreter for compile-time strings)

s3 = "hello" + "_world"
print(s1 is s3)   # May be False or True depending on optimization (usually True due to interning)

# Explicit interning using sys.intern()
import sys
s4 = sys.intern("dynamic string!")
s5 = sys.intern("dynamic string!")
print(s4 is s5)   # True
True
True
True

Interning saves memory by reusing immutable string objects. Python automatically interns some strings.
