📖 Memory Management in Python

What is Memory Management? — Memory management is the process of controlling and coordinating how a computer program allocates, uses, and frees memory. In Python, memory management is automatic — developers don't need to manually allocate or deallocate memory. Python uses a private heap to store all objects and data structures.

🗃️ Stack Memory

  • Stores static data — function calls, local variables, and references to heap objects.
  • Managed in LIFO (Last-In-First-Out) order.
  • Fast access but limited size (stack overflow risk).
  • Each thread gets its own stack.
  • Variables are automatically deallocated when function exits.

🏗️ Heap Memory

  • Stores all Python objects — lists, dicts, instances, strings, etc.
  • Managed by Python's memory manager (allocator).
  • Slower than stack but dynamic & flexible.
  • Objects are accessed via references (pointers) stored in stack.
  • Heap memory is cleaned by Garbage Collector when no longer referenced.

🔧 How Python Handles Memory

  • Private Heap: All objects live in a private heap managed by Python Memory Manager.
  • Reference Counting: Primary mechanism — each object tracks how many references point to it. When count reaches 0, memory is freed immediately.
  • Garbage Collector (GC): Handles circular references (objects referencing each other). Uses generational garbage collection (generations 0, 1, 2) for efficiency.
  • Memory Pools (Arenas): For objects smaller than 512 bytes, Python uses pre-allocated pools to reduce fragmentation and speed up allocation.
  • Small Integer Caching: Integers from -5 to 256 are pre-allocated (singletons) for performance.
  • String Interning: Python may reuse immutable string objects to save memory (especially for identifiers and compile-time strings).

🗑️ Garbage Collection (GC) Details

Python's GC is supplemental to reference counting. It detects and collects reference cycles (e.g., two objects referencing each other).

  • gc.collect() — manually trigger collection.
  • gc.get_threshold() — returns (gen0, gen1, gen2) thresholds (default 700, 10, 10).
  • gc.disable() / gc.enable() — control GC behavior.

🔗 String Interning & Caching

Python reuses immutable objects to save memory:

  • Small ints (-5..256) — always pre-cached.
  • Compile-time strings — automatically interned.
  • sys.intern() — manually intern strings for memory optimization.
  • Interning reduces duplication but increases lookup cost.
📝 Quick Example: Reference Counting 
import sys
x = [1, 2, 3]
# Reference count = 1
y = x                # now count = 2
print(sys.getrefcount(x) - 1)   # Output: 2
# Deleting references
del y                # count goes back to 1
del x                # count = 0 → object freed

🎯 Key Takeaways

  • Python uses automatic memory management — no manual malloc/free.
  • Stack holds references and primitive data; Heap holds actual objects.
  • Reference counting handles most memory cleanup instantly.
  • Generational GC handles circular references efficiently.
  • Interning & Caching optimize memory for small ints and strings.
  • Use sys.getrefcount(), id(), and gc module to inspect memory behavior.
Understanding memory management helps write memory-efficient Python code and avoid leaks.
