Higher-Order Functions

✨ What are Higher-Order Functions? — Functions that can accept other functions as arguments or return functions. map(), filter(), and reduce() are classic functional programming tools in Python that enable declarative data transformation without explicit loops.

📦 map() → Transform

Purpose: Applies a function to every item in an iterable and returns an iterator of transformed results.

map(function, iterable) 
# Example: square all numbers
nums = [1, 2, 3, 4]
squared = list(map(lambda x: x**2, nums))
# Result: [1, 4, 9, 16]
Use when: You need to transform every element

🔍 filter() → Select

Purpose: Filters elements based on a predicate function (returns True/False). Keeps only elements where function returns True.

filter(predicate, iterable) 
# Example: keep even numbers
nums = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda x: x % 2 == 0, nums))
# Result: [2, 4, 6]
Use when: You need to select a subset based on condition

➕ reduce() → Combine

Purpose: Cumulatively applies a function to items, reducing the iterable to a single value. (from functools import reduce)

reduce(function, iterable[, initializer]) 
# Example: sum all numbers
from functools import reduce
nums = [1, 2, 3, 4]
total = reduce(lambda a, b: a + b, nums)
# Result: 10
Use when: You need to aggregate/combine into single value

🔬 Real Example: Same Data with map/filter/reduce

Operation Code Result
map() map(lambda x: x*2, [1,2,3]) → [2, 4, 6]
filter() filter(lambda x: x > 2, [1,2,3,4]) → [3, 4]
reduce() reduce(lambda a,b: a*b, [1,2,3,4]) → 24 (1*2*3*4)

⚡ Performance Notes

  • map() and filter() return iterators (lazy evaluation) in Python 3 → memory efficient.
  • Use list() to materialize results if needed.
  • reduce() is less common; often sum(), max(), or list comprehensions are more readable.
  • List comprehensions are often more Pythonic than map/filter for simple cases.

🔄 Comparison with Comprehensions

# map vs list comp
map(lambda x: x**2, nums) ↔ [x**2 for x in nums]

# filter vs list comp
filter(lambda x: x>0, nums) ↔ [x for x in nums if x>0]

Data cleaning (map) Remove invalid entries (filter) Aggregations (reduce) ETL pipelines
